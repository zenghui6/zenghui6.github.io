简单工厂模式

该模式对对象创建管理方式最为简单,该模式通过向工厂传递类型来指定要创建的对象,其UML类图如下:

Car接口

public interface Car {
    void make();
}

各种车实现接口

public class BMW implements Car {
    @Override
    public void make() {
        System.out.println("宝马!!");
    }
}
public class Wuling implements Car {
    @Override
    public void make() {
        System.out.println("五菱!!");
    }
}

车工厂

public class CarFactory {
    private Car getCar(String carName){
        if (carName.equals("Wuling")){
            return new Wuling();
        }else if (carName.equals("BMW")){
            return new BMW();
        }
        return null;
    }
}

工厂方法模式

和简单工厂模式中工厂负责生产所有产品相比，工厂方法模式将生成具体产品的任务分发给具体的产品工厂,每辆车都要自己的工厂，其UML类图如下：

将工厂抽象出来(成为一种规范), 每个产品自己去实现自己的工厂

IFactory接口: 生成车的工厂

public interface IFactory {
    Car makeCar();
}

ICar接口: 车的接口

public interface Car {
    void make();
}

车的实现

public class BMW implements Car {
    @Override
    public void make() {
        System.out.println("宝马!!");
    }
}
public class Wuling implements Car {
    @Override
    public void make() {
        System.out.println("五菱!!");
    }
}

每种车都有自己的工厂

public class BMWFactory implements IFactory {
    @Override
    public Car makeCar() {
        return new BMW();
    }
}

抽象工厂模式

抽象工厂是一个超级工厂,叫做工厂的工厂

用来规范工厂的行为.

抽象工厂模式易于增加产品的内部规模，工厂方法模式易于增加一个产品的种类。
